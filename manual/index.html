This software allows two people to look into each other's PC screen. The main feature lies in the ability for either or both the caller or the answerer to interact with the remote's PC.
I wrote this because family's connection in Italy is firewalled by the ISP. Making it impossible for me to connect to their PCs to do administer them.
This software is very simple and it doesn't attempt to do anything behind the users' back. It does not contain any spyware, nagware or whatever.

Quick Start

BIG FAT WARNING !
Use a strong password and be really really sure that the remote party is indeed someone that you can trust !!!

You are either a caller or a answerer.

For the caller (example: the family behind firewall):

1) Run DSharingu.
2) Click on the button Settings...
3) Insert the destination IP (given to you by the answerer*).
3) Choose if you want to show your screen (on by default, leave it like that).
4) Choose if you want to share the control of your PC (on by default, leave it like that but make sure you can trust the remote party).
5) Set a password that you'll give to the remote.
6) Choose if you want to see the remote's desk (on by default, set it to off).
7) Click OK to close the Settings dialog.
8) Click on Call... and hopefully the connection will go through.


For the answerer (the family computer wiz that wants to help the family):

1) Run DSharingu.
2) Click on the button Settings... you are going to get called, so, no need to enter any IP here.
3) Choose if you want to show and/or share your screen (on by default, you probably want to change this to off)
4) Choose if you want to see the remote's screen (off by default, you probably want to change this to on).
5) Give your IP to the family *
6) Patiently wait for the call...

This is only one scenario of course. One can play with the settings at will, but I generally just want to cover the case the program has been designed for.

* Personally I user DyDNS. It's very useful to have the less computer-literate people call you at a fixed readable address, rather than a numeric IP !

Known issues

- Currently, interaction is only possible throught mouse . Keyboard support will come later.
- When opening the About and Settings dialogs of the remote instance of the program, images will stop coming (modal dialogs stop the main gameflow, will fix in next version).
- Very high bandwidth is still required.

Requirements for a smooth usage

- Windows XP/2000 (currently only tested on Windows XP).
- A decent graphic chip with basic OpenGL support (better if you have recent video-card drivers anyway !).
- A non-ancient system: 1GHz CPU and 512MB of RAM.
- When running through the Internet, DSL/Cable connection or better.
